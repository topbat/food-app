# AI 服务（ai-service）测试指南

> Python FastAPI 微服务，端口 **8086**，接口前缀 `/api/ai`，全部公开（无需鉴权）。
> 统一响应体：`{"code":0,"message":"操作成功","data":...}`；参数错误 `code=40000`；服务异常 `code=50000`。

## 一、安装依赖

```powershell
cd d:\self\food-app\backend\ai-service
pip install -r requirements.txt
```

## 二、启动服务

```powershell
python main.py
```

- 默认环境 **dev**（日志级别 DEBUG、热重载开启、CORS 仅放行 `http://localhost:5173`）
- 默认监听 `0.0.0.0:8086`
- 日志同时输出到控制台与 `logs/ai-service.log`（10MB×5 滚动）
- 启动日志会打印当前环境与端口，可据此确认环境是否正确

### 切换环境

```powershell
# UAT 环境（日志 INFO、关闭热重载）
$env:APP_ENV = 'uat'; python main.py

# 生产环境
$env:APP_ENV = 'prod'; python main.py

# 恢复默认 dev
Remove-Item Env:APP_ENV
```

## 三、接口测试

> **Windows PowerShell 5.1 中文显示说明（实测）**：以下 `Invoke-RestMethod` 命令发送中文请求体均正常（服务端可正确命中规则），但响应中的中文在控制台可能显示为乱码（如 `æä½æå`），这只是 PS 5.1 的显示问题，实际字节为正确的 UTF-8。如需查看中文原文，可改用按 UTF-8 字节读取的写法：
>
> ```powershell
> $json = '{"ingredients":[{"name":"五花肉","amount":200}],"scene":"减脂"}'
> $bytes = [Text.Encoding]::UTF8.GetBytes($json)
> $resp = Invoke-WebRequest -Uri "http://localhost:8086/api/ai/substitute" -Method Post -ContentType "application/json; charset=utf-8" -Body $bytes -UseBasicParsing
> [Text.Encoding]::UTF8.GetString($resp.RawContentStream.ToArray())
> ```
>
> PowerShell 7+（pwsh）无此问题。另：若 `localhost` 被解析为 IPv6 导致连接失败，改用 `127.0.0.1`。

### 3.2 智能替换 POST /api/ai/substitute

**用例 1：减脂场景命中规则（五花肉→鸡胸肉）**

```powershell
$body = @{ ingredients = @(@{ name = "五花肉"; amount = 200 }, @{ name = "白糖"; amount = 10 }); scene = "减脂" } | ConvertTo-Json -Depth 5
Invoke-RestMethod -Uri "http://localhost:8086/api/ai/substitute" -Method Post -ContentType "application/json; charset=utf-8" -Body $body | ConvertTo-Json -Depth 5
```

curl 写法：

```bash
curl -X POST http://localhost:8086/api/ai/substitute -H "Content-Type: application/json" -d "{\"ingredients\":[{\"name\":\"五花肉\",\"amount\":200}],\"scene\":\"减脂\"}"
```

预期响应（节选）：

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "suggestions": [
      { "sourceName": "五花肉", "targetName": "鸡胸肉", "reason": "……", "calorieSavedPer100g": 395 },
      { "sourceName": "白糖", "targetName": "木糖醇", "reason": "……", "calorieSavedPer100g": 160 }
    ]
  }
}
```

**用例 2：未命中任何规则**

```powershell
$body = @{ ingredients = @(@{ name = "黄瓜"; amount = 100 }); scene = "减脂" } | ConvertTo-Json -Depth 5
Invoke-RestMethod -Uri "http://localhost:8086/api/ai/substitute" -Method Post -ContentType "application/json; charset=utf-8" -Body $body | ConvertTo-Json -Depth 5
```

预期：`code=0`，`data.suggestions` 为空数组，`message` 为提示文案「当前食材在该场景下暂无替换建议…」。

**用例 3：未知场景**

scene 传 `"增肌"` → `code=0`、`suggestions=[]`、`message` 提示当前支持的场景列表（减脂、控糖、素食、低嘌呤）。

### 3.3 自由文本解析 POST /api/ai/parse-recipe

**用例 1：正常文本**

```powershell
$body = @{ freeText = "宫保鸡丁。鸡胸肉150g切丁，生抽10ml腌制15分钟。黄瓜与胡萝卜洗净去皮。黄瓜胡萝卜切丁，大葱切节。热锅冷油，大火爆香干辣椒，下鸡丁翻炒5分钟。盛入深盘，撒花生米点缀。" } | ConvertTo-Json -Depth 5
Invoke-RestMethod -Uri "http://localhost:8086/api/ai/parse-recipe" -Method Post -ContentType "application/json; charset=utf-8" -Body $body | ConvertTo-Json -Depth 8
```

curl 写法：

```bash
curl -X POST http://localhost:8086/api/ai/parse-recipe -H "Content-Type: application/json" -d "{\"freeText\":\"番茄炒蛋。番茄洗净。番茄切块。大火翻炒3分钟。装盘撒葱花。\"}"
```

预期响应要点：

- `data.recipe_name` = "宫保鸡丁"（第一句作为菜名）
- `data.recipe_id` 为 uuid 字符串
- `data.steps` 按 前期准备→洗→切→煮→装盘 排序：
  - 含「腌制」的句子 → 前期准备，且提取 `timer_minutes: 15`
  - 含「洗净」的句子 → 洗
  - 含「切丁/切节」的句子 → 切
  - 含「翻炒/大火」的句子 → 煮，且 `fire_control: "大火"`、`timer_minutes: 5`
  - 含「盘/撒/点缀」的句子 → 装盘
- `data.nutrition_per_serving` 四项均为 0（占位，需人工/模型补全）

**边界用例 2：空文本**

```powershell
$body = @{ freeText = "" } | ConvertTo-Json
Invoke-RestMethod -Uri "http://localhost:8086/api/ai/parse-recipe" -Method Post -ContentType "application/json; charset=utf-8" -Body $body
```

预期：`{"code":40000,"message":"freeText 不能为空，请提供菜谱自由文本","data":null}`

**边界用例 3：纯英文文本**

```powershell
$body = @{ freeText = "Kung Pao Chicken. Dice the chicken. Stir fry for 5 minutes." } | ConvertTo-Json
Invoke-RestMethod -Uri "http://localhost:8086/api/ai/parse-recipe" -Method Post -ContentType "application/json; charset=utf-8" -Body $body | ConvertTo-Json -Depth 8
```

预期：不报错，`code=0`；第一句作为菜名；英文句子不含中文关键词，全部默认归类「前期准备」。

**边界用例 4：请求体格式错误**

发送非 JSON 或缺字段（如 `{"foo":1}` 时 freeText 取默认空串 → 40000；非法 JSON → 40000「参数错误」）。

## 四、日志验证

- 每次请求后控制台与 `logs/ai-service.log` 应出现访问日志：`访问日志：POST /api/ai/substitute -> 200 耗时=xx.xxms`
- 规则命中 / 阶段识别等关键分支输出中文 INFO 日志，例如：
  - `替换规则命中：场景=减脂，食材「五花肉」→「鸡胸肉」…`
  - `阶段识别命中：关键词「炒」→ 阶段「煮」…`

## 五、异常场景

任何未捕获异常对外统一返回 `{"code":50000,"message":"服务繁忙，请稍后重试"}`，堆栈仅写入日志文件，不泄露给调用方。
