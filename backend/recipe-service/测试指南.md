# 菜谱结构化服务（recipe-service）测试指南

## 一、模块简介

- **职责**：菜谱分页查询（含冰箱清理模式）、五步法详情、步骤平铺（供厨房服务调用）、智能食材替换、浏览计数、UGC 上传、食材库与标签字典。
- **端口**：`8082`，接口前缀 `/api/recipe`。
- **表前缀**：`recipe_`，共 7 张表：`recipe_info`（菜谱主表）、`recipe_step`（五步法步骤）、`recipe_ingredient`（菜谱食材）、`recipe_ingredient_lib`（基础食材库）、`recipe_tag`（标签字典）、`recipe_tag_relation`（标签关联）、`recipe_substitute_rule`（替换规则）。
- **鉴权**：除 `POST /api/recipe/ugc` 需登录外全部公开；token 由 user-service 签发，两服务共享同一 JWT 密钥（dev 默认值相同，可直接互通）。
- **统一响应体**：`{"code":0,"message":"操作成功","data":...,"timestamp":"..."}`；分页结构 `{"total":N,"page":1,"size":10,"list":[...]}`。

## 二、环境准备

| 环境 | 数据库 | 说明 |
|---|---|---|
| dev（默认） | H2 内存库（MODE=MySQL） | **零外部依赖**：JPA 自动建表（`ddl-auto: update` + `defer-datasource-initialization: true`）后自动执行 `src/main/resources/data.sql` 灌入种子数据 |
| uat | MySQL 8.0 `localhost:3306/food_app` | 需先执行 `sql/00_database.sql` 与 `sql/02_recipe.sql` 建库建表灌数（`ddl-auto: none`）；`DB_USERNAME`/`DB_PASSWORD` 默认 root/root，JWT 密钥用 `JWT_SECRET` |
| prod | MySQL 8.0（同 uat） | 另外关闭 SQL 打印、日志级别 INFO |

切换 profile（默认 dev，无需任何参数）：

```powershell
mvn -pl recipe-service spring-boot:run "-Dspring-boot.run.profiles=uat"
# 或：$env:SPRING_PROFILES_ACTIVE = "uat"
```

## 三、启动方式与成功标志

```powershell
cd d:\self\food-app\backend
# 首次启动前需安装父 POM 与 common（仅一次）：
mvn -N install; mvn -pl common install -DskipTests
# 启动（-am 自动连带构建 common）：
mvn -pl recipe-service -am spring-boot:run
```

**启动成功标志**（看控制台或 `logs/recipe-service.log`）：

- `Tomcat started on port 8082`
- `Started RecipeServiceApplication in X seconds`
- dev 下 data.sql 执行无报错（启动日志中无 `ScriptStatementFailedException`），用 `GET /api/recipe/list` 返回 `total:10` 验证种子数据就绪

> **PowerShell 中文请求体注意（实测）**：Windows 控制台默认 GBK 代码页，`-d` 内联中文（如 `减脂`）会以 GBK 字节发出，服务端返回 `{"code":50000}`（JSON parse error: Invalid UTF-8）。解决：
> ① 测试前先执行 `chcp 65001`；
> ② 或将请求体保存为 **UTF-8 文件** 后用 `curl.exe --data-binary "@body.json"` 发送（最稳妥）。
> 下文统一用 `curl.exe`（PowerShell 中 `curl` 是 `Invoke-WebRequest` 别名）；URL 中的中文 query 参数已做 URL 编码可直接复制执行。

### 4. POST /api/recipe/{id}/substitute — 智能替换

```powershell
# 先把请求体存为 UTF-8 文件（PowerShell 5.1 默认 UTF-16，必须指定 utf8）：
'{"scene":"减脂"}' | Out-File -Encoding utf8 sub.json
curl.exe -s -X POST "http://localhost:8082/api/recipe/1/substitute" -H "Content-Type: application/json" --data-binary "@sub.json"
```

预期响应（实测，宫保鸡丁 + 减脂 → 白糖→木糖醇）：

```json
{
  "code": 0,
  "data": {
    "originalCalories": 350.00,
    "newCalories": 342.0,
    "calorieDiff": 8.0,
    "substitutions": [
      { "sourceName": "白糖", "targetName": "木糖醇",
        "reason": "木糖醇热量约为蔗糖的60%，减脂期可有效减少精制糖摄入", "calorieDiff": 8.0 }
    ]
  }
}
```

其他可验证组合：菜谱 3+控糖（白糖→木糖醇）、菜谱 2+素食（猪肉末→豆腐）、菜谱 9+减脂（五花肉→鸡胸肉）。
要点：场景仅 减脂/控糖/素食/低嘌呤；`calorieDiff` 为正表示减少；仅"克/毫升/g/ml"单位的食材参与热量换算，其他单位只给建议（diff=0）。

### 6. POST /api/recipe/ugc — UGC 上传（需登录）

先到 user-service（:8081）拿 token（dev 下两服务 JWT 密钥相同，token 通用）：

```powershell
$token = (Invoke-RestMethod -Method Post -Uri "http://localhost:8081/api/user/login" -ContentType "application/json" -Body '{"username":"demo","password":"123456"}').data.token
```

请求体保存为 UTF-8 文件 `ugc.json`（五步法 5 阶段必须齐全；13=西红柿、12=鸡蛋）：

```json
{"title":"家常西红柿炒蛋","cuisineType":9,"difficulty":1,"totalTimeMin":10,"servings":1,"description":"国民快手菜","ingredients":[{"ingredientId":13,"amount":200,"unit":"克","isEssential":1},{"ingredientId":12,"amount":2,"unit":"个","isEssential":1}],"steps":[{"phase":"PREPARE","stepIndex":1,"actionTitle":"备料","detail":"西红柿200克、鸡蛋2个"},{"phase":"WASH","stepIndex":1,"actionTitle":"清洗","detail":"西红柿洗净"},{"phase":"CUT","stepIndex":1,"actionTitle":"切块","detail":"西红柿切月牙块"},{"phase":"COOK","stepIndex":1,"actionTitle":"炒制","detail":"先炒蛋后下西红柿，翻炒2分钟","timerSec":120,"firePower":"大火"},{"phase":"PLATE","stepIndex":1,"actionTitle":"装盘","detail":"撒葱花出锅"}],"suitableTagIds":[12],"unsuitableTagIds":[]}
```

```powershell
curl.exe -s -X POST "http://localhost:8082/api/recipe/ugc" -H "Content-Type: application/json" -H "Authorization: Bearer $token" --data-binary "@ugc.json"
```

预期（实测）：`{"code":0,"message":"菜谱已提交审核","data":{"id":11}}`
要点：新菜谱 `status=2`（待审核）、`sourceType=2`，**不出现**在 /list 与 /{id} 查询中（仍返回 total=10 / 40400）；热量按食材库自动计算并折算单人份（"个"等单位不参与计算）。

## 六、异常场景用例（以实际代码为准）

| 场景 | 操作 | 预期 code | 预期 message |
|---|---|---|---|
| 菜谱不存在 | GET /api/recipe/999（/steps、/substitute 同理） | 40400 | 菜谱不存在 |
| 菜谱待审核/下架（status≠1） | GET /api/recipe/11（刚上传的 UGC） | 40400 | 菜谱不存在或已下架 |
| UGC 未带 token | POST /ugc 无 Authorization（HTTP 401） | 40100 | 未登录或登录已过期 |
| UGC 缺五步法阶段 | steps 缺 PLATE | 40000 | 五步法不完整，缺少「装盘」阶段 |
| UGC 食材不存在 | ingredientId=999 | 40400 | 食材不存在: ID=999 |
| UGC 参数校验 | title 为空 / cuisineType=10 / difficulty=4 | 40000 | 含具体字段提示（如 `cuisineType 菜系编码必须在1-9之间`） |
| substitute 场景为空 | body `{"scene":""}` | 40000 | scene 场景不能为空 |
| substitute 无规则命中 | 菜谱 10 + 任意场景 | 0 | substitutions 为空数组、calorieDiff=0（非报错） |
| 请求体非 UTF-8 中文 | GBK 编码 JSON（HTTP 200） | 50000 | 服务内部错误，请稍后重试 |

实测示例：

```powershell
# 菜谱不存在 → 40400
curl.exe -s "http://localhost:8082/api/recipe/999"
# UGC 无 token → 40100
curl.exe -s -X POST "http://localhost:8082/api/recipe/ugc" -H "Content-Type: application/json" -d '{\"title\":\"x\"}'
```

## 八、数据库验证点

**dev（H2 控制台）**：浏览器访问 `http://localhost:8082/h2-console`，JDBC URL 填 `jdbc:h2:mem:recipedb`，用户名 `sa`，密码留空。

```sql
-- 各表种子条数（预期 36 / 16 / 10 / 73 / 77 / 37 / 7）
SELECT (SELECT COUNT(*) FROM recipe_ingredient_lib)  AS lib,
       (SELECT COUNT(*) FROM recipe_tag)             AS tag,
       (SELECT COUNT(*) FROM recipe_info)            AS info,
       (SELECT COUNT(*) FROM recipe_step)            AS step,
       (SELECT COUNT(*) FROM recipe_ingredient)      AS ri,
       (SELECT COUNT(*) FROM recipe_tag_relation)    AS rel,
       (SELECT COUNT(*) FROM recipe_substitute_rule) AS sub;

-- 每道菜五步法覆盖（phases 应全为 5）
SELECT recipe_id, COUNT(DISTINCT phase) AS phases, COUNT(*) AS steps
FROM recipe_step GROUP BY recipe_id ORDER BY recipe_id;

-- 浏览计数验证（调 /view 后 +1）
SELECT id, title, view_count FROM recipe_info WHERE id = 1;

-- UGC 上传后：status=2 待审核、source_type=2、作者ID
SELECT id, title, status, source_type, author_id, calories_kcal FROM recipe_info WHERE id > 10;
```

**uat/prod（MySQL）**：`mysql -uroot -p food_app` 后执行同样 SQL（表结构与种子数据见 `sql/02_recipe.sql`）。

验证要点：

1. UGC 上传后 `recipe_info` 新增 status=2 行，`recipe_step` 新增 5 行、`recipe_ingredient` 新增 2 行、`recipe_tag_relation` 新增 1 行（suitableTagIds 去重）。
2. /view 调用次数与 `view_count` 增量一致（原子 UPDATE 不丢并发计数）。
3. H2 为内存库，**重启即清空重灌**种子数据。
