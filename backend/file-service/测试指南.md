# 文件存储服务 file-service 测试指南

> 端口 **8087**，前缀 `/api/file`，**全部接口需鉴权**（请求头 `Authorization: Bearer <token>`）。
> 默认存储为 MinIO（`foodapp.storage.type=minio`），可切换阿里云 OSS。
> dev 环境数据库为 H2 内存库，重启即清空；uat/prod 用 MySQL（先执行 `sql/06_file.sql`）。

## 一、MinIO 安装与启动

文件服务启动**不依赖** MinIO（懒连接），但上传文件需要 MinIO 在 9000 端口运行。两种方式任选：

**方式一：Docker**

```powershell
docker run -p 9000:9000 -p 9001:9001 minio/minio server /data --console-address ":9001"
```

**方式二：Windows 直接下载 minio.exe（免 Docker）**

```powershell
Invoke-WebRequest https://dl.min.io/server/minio/release/windows-amd64/minio.exe -OutFile minio.exe
.\minio.exe server D:\minio-data --console-address ":9001"
```

- 默认账号密码：`minioadmin` / `minioadmin`
- API 端口 `9000`；**Web 控制台** http://localhost:9001 （用上述账号登录，可在 Object Browser 中查看 `food-app` 桶里上传的文件与缩略图）
- bucket `food-app` **无需手动创建**：文件服务首次上传时自动创建并设置公共读策略（匿名可 GET）

## 二、启动服务

```powershell
cd d:\self\food-app\backend
mvn -q -N install; mvn -q -pl common install -DskipTests   # 首次必须
mvn -pl file-service spring-boot:run                        # dev 环境（H2 + 本机 MinIO）
```

启动成功标志：日志出现 `Tomcat started on port 8087`。

## 三、前置：获取 token

文件服务全部接口需要登录，先到用户服务（8081）注册/登录拿 token：

```powershell
# 如 8081 未启动：mvn -pl user-service spring-boot:run
$reg = Invoke-RestMethod -Method Post -Uri http://localhost:8081/api/user/register `
  -ContentType 'application/json;charset=utf-8' `
  -Body '{"username":"filetester","password":"123456","nickname":"文件测试员"}'
$token = $reg.data.token
# 已注册过则改用登录：
# $login = Invoke-RestMethod -Method Post -Uri http://localhost:8081/api/user/login -ContentType 'application/json;charset=utf-8' -Body '{"username":"filetester","password":"123456"}'
# $token = $login.data.token
$headers = @{ Authorization = "Bearer $token" }
```

## 四、上传用例

### 4.1 生成一张测试 PNG（System.Drawing 画 100x100 色块）

```powershell
Add-Type -AssemblyName System.Drawing
$bmp = New-Object System.Drawing.Bitmap 100,100
$g = [System.Drawing.Graphics]::FromImage($bmp)
$g.Clear([System.Drawing.Color]::Orange)
$bmp.Save("$env:TEMP\test.png", [System.Drawing.Imaging.ImageFormat]::Png)
$g.Dispose(); $bmp.Dispose()
```

### 4.2 上传图片（PowerShell 5.1 无 -Form，推荐 curl.exe -F）

```powershell
curl.exe -s -X POST http://localhost:8087/api/file/upload `
  -H "Authorization: Bearer $token" `
  -F "file=@$env:TEMP\test.png" `
  -F "bizType=recipe"
```

> PowerShell 7+ 可用：`Invoke-RestMethod -Method Post -Uri http://localhost:8087/api/file/upload -Headers $headers -Form @{ file = Get-Item "$env:TEMP\test.png"; bizType = 'recipe' }`

预期响应（code=0）：

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "id": 1,
    "url": "http://localhost:9000/food-app/recipe/2026/06/12/<uuid>.png",
    "thumbUrl": "http://localhost:9000/food-app/recipe/2026/06/12/<uuid>_thumb.jpg",
    "originalName": "test.png",
    "contentType": "image/png",
    "sizeBytes": 312,
    "storageType": "minio",
    "bizType": "recipe"
  }
}
```

**对象键命名约定（前端依赖）**：`{bizType}/{yyyy/MM/dd}/{uuid}.{ext}`；缩略图为同路径 `{uuid}_thumb.jpg`，即 **原 url 去扩展名 + `_thumb.jpg`** 可直接推导。

### 4.3 上传视频（mp4/mov，≤100MB）

```powershell
curl.exe -s -X POST http://localhost:8087/api/file/upload `
  -H "Authorization: Bearer $token" `
  -F "file=@D:\某个视频.mp4" `
  -F "bizType=post"
```

预期：`url` 为视频地址，`thumbUrl` 为视频第一帧缩放成的 400px JPG。若视频编码 jcodec 不支持（如 H.265），`thumbUrl` 为 `null` 且服务日志出现 `[缩略图] 视频缩略图生成失败` 的 warn —— 不阻断上传，属预期行为。

### 4.4 缩略图验证

```powershell
$resp = (curl.exe -s -X POST http://localhost:8087/api/file/upload -H "Authorization: Bearer $token" -F "file=@$env:TEMP\test.png") | ConvertFrom-Json
(Invoke-WebRequest $resp.data.url).StatusCode        # 预期 200（原图可匿名访问）
(Invoke-WebRequest $resp.data.thumbUrl).StatusCode   # 预期 200（缩略图可匿名访问，宽400px JPG）
# 推导规则验证：原 url 去扩展名 + _thumb.jpg == thumbUrl
```

### 4.5 我的上传记录分页

```powershell
Invoke-RestMethod -Uri 'http://localhost:8087/api/file/my?page=1&size=10' -Headers $headers | ConvertTo-Json -Depth 5
```

预期：`data: {total,page,size,list:[{id,url,thumbUrl,originalName,contentType,sizeBytes,storageType,bizType,createdAt}]}`，按上传时间倒序。

## 五、异常用例

| 用例 | 操作 | 预期 |
|---|---|---|
| 不支持的类型 | 上传 `.txt`/`.exe` 等 | `code=40000`，message 含「不支持的文件类型」 |
| 图片超限 | 上传 >10MB 的 jpg/png | `code=40000`，「图片大小不能超过 10MB」 |
| 视频超限 | 上传 >100MB 的 mp4 | `code=40000`（>100MB 由 Spring multipart 限制拦截，统一转 40000 友好提示） |
| 非法业务类型 | `-F "bizType=hack"` | `code=40000`，「不支持的业务类型」 |
| 未带 token | 不加 Authorization 头 | HTTP 401，`code=40100`「未登录或登录已过期」 |
| MinIO 未启动 | 停掉 MinIO 后上传 | `code=50000`，「**对象存储服务不可用，请先启动 MinIO**」（服务本身不受影响） |

示例：

```powershell
# 不支持的类型 → 40000
'hello' | Out-File "$env:TEMP\test.txt" -Encoding ascii
curl.exe -s -X POST http://localhost:8087/api/file/upload -H "Authorization: Bearer $token" -F "file=@$env:TEMP\test.txt"

# 未带 token → 40100
curl.exe -s -X POST http://localhost:8087/api/file/upload -F "file=@$env:TEMP\test.png"
```

## 六、切换阿里云 OSS

文件服务采用**存储策略模式**，业务接口与响应结构不变，仅改配置：

```yaml
foodapp:
  storage:
    type: aliyun                                  # minio（默认）→ aliyun
    aliyun:
      endpoint: oss-cn-hangzhou.aliyuncs.com      # 你的地域 endpoint
      bucket: your-bucket                          # 需开通「公共读」
      access-key-id: ${OSS_ACCESS_KEY_ID}          # 环境变量注入，禁止硬编码
      access-key-secret: ${OSS_ACCESS_KEY_SECRET}
```

uat/prod 环境直接用环境变量：`STORAGE_TYPE=aliyun`、`OSS_ENDPOINT`、`OSS_BUCKET`、`OSS_ACCESS_KEY_ID`、`OSS_ACCESS_KEY_SECRET`（MinIO 对应 `MINIO_ENDPOINT`、`MINIO_ACCESS_KEY`、`MINIO_SECRET_KEY`、`MINIO_BUCKET`）。
切换后访问 URL 形如 `https://{bucket}.{endpoint}/{objectKey}`，对象键与缩略图推导规则完全一致。

## 七、数据库验证

**dev（H2 控制台）**：浏览器访问 http://localhost:8087/h2-console ，JDBC URL 填 `jdbc:h2:mem:filedb`，用户名 `sa`，密码留空：

```sql
SELECT id, user_id, biz_type, object_key, url, thumb_url, content_type, size_bytes, storage_type, created_at
FROM file_record ORDER BY id DESC;
```

**uat/prod（MySQL）**：先执行 `sql/06_file.sql` 建表后查询同上（库 `food_app`，表 `file_record`）。
