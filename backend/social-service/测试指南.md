# 社交与UGC服务（social-service）测试指南

## 1. 模块简介

| 项目 | 说明 |
|---|---|
| 服务名 | social-service（社交与UGC服务） |
| 端口 | **8085** |
| 接口前缀 | `/api/social` |
| 表前缀 | `social_`（social_post / social_comment / social_rating / social_checkin / social_badge / social_user_badge / social_like_record） |
| 核心职责 | 评论（含菜谱步骤级评论与楼中楼）、作品墙（发帖/列表/详情）、点赞 toggle、菜谱评分、烹饪打卡与连续天数、成就徽章、分享卡数据 |
| 技术 | Spring Boot 3.3.5 + Java 21 + JPA |

**鉴权策略**：拦截 `/api/social/**`，白名单（公开）仅 4 个查询接口：

- `GET /api/social/comment/list`（评论列表）
- `GET /api/social/post/list`（作品墙列表）
- `GET /api/social/post/{id}`（帖子详情，`/api/social/post/*` 只放行一级路径，不会放行 POST 发帖）
- `GET /api/social/rating/{recipeId}`（评分查询）

其余（发评论/发帖/点赞/评分/打卡/打卡月历/徽章墙/分享卡）一律需要 `Authorization: Bearer <token>`，缺失或非法返回 HTTP **401** / code **40100**。

**对其他服务的依赖（启动顺序）**：

1. **用户服务 :8081（展示依赖 + 发 Token）**——评论/帖子列表与分享卡需补充昵称头像，调用 `GET /api/user/public/{id}`，结果**内存缓存 10 分钟**；用户服务不可用时**降级**为昵称"美食家"、头像 null（列表可用性优先，不报错，降级结果不写缓存，便于下游恢复后自动回源）。
2. **菜谱服务 :8082（无直接调用）**——评论/评分/打卡中的 `recipeId` 仅作为关联字段存储，本服务**不校验菜谱是否存在**；联调展示时前端才会用到菜谱服务。

> 推荐启动顺序：用户服务 8081 → 社交服务 8085（菜谱服务非必需）。

## 4. 前置条件：获取 Token

```powershell
# 注册（已注册过则改用下面的登录）
$reg = Invoke-RestMethod -Uri "http://localhost:8081/api/user/register" -Method Post -ContentType "application/json;charset=utf-8" -Body (@{username="social_tester";password="Test@1234";nickname="社交测试员"} | ConvertTo-Json)
$token = $reg.data.token

# 登录
$login = Invoke-RestMethod -Uri "http://localhost:8081/api/user/login" -Method Post -ContentType "application/json;charset=utf-8" -Body (@{username="social_tester";password="Test@1234"} | ConvertTo-Json)
$token = $login.data.token

$H = @{ Authorization = "Bearer $token" }
$token   # 确认非空
```

curl.exe 等价：

```powershell
curl.exe -s -X POST http://localhost:8081/api/user/login -H "Content-Type: application/json" -d "{\"username\":\"social_tester\",\"password\":\"Test@1234\"}"
# 从响应 data.token 复制，后续命令加 -H "Authorization: Bearer <token>"
```

> 列表中昵称头像来自用户服务 `/api/user/public/{id}`：用户服务在线时显示"社交测试员"；离线时降级为"美食家"。

## 5. 接口测试用例（共 12 个接口）

统一响应：`{"code":0,"message":"...","data":...,"timestamp":"..."}`；分页统一 `{total,page,size,list}`。以下命令均假设已设置 `$H`。

### 5.1 POST /api/social/post —— 发帖（鉴权）

```powershell
$post = Invoke-RestMethod -Uri "http://localhost:8085/api/social/post" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{
  recipeId = 1
  content  = "第一次做宫保鸡丁，糊辣香拉满！"
  imageUrls = @("https://picsum.photos/seed/p1/600/400","https://picsum.photos/seed/p2/600/400")
  postType = 1
} | ConvertTo-Json)
$post | ConvertTo-Json -Depth 5
$postId = $post.data.id
```

预期：code=0，data=`{"id":1,"newBadges":[{"badgeName":"晒盘新秀","icon":"📸"}]}`（首帖触发 POST_COUNT=1 徽章；累计 5 帖触发"发帖达人"）。
约束：content 必填 ≤1000 字；postType 必填 1作品晒图/2烹饪打卡/3美食日记；recipeId、imageUrls 可空（imageUrls 序列化为 JSON 数组字符串存 TEXT 列）。

### 5.2 GET /api/social/post/list —— 作品墙（公开）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/post/list?page=1&size=10" | ConvertTo-Json -Depth 6
Invoke-RestMethod -Uri "http://localhost:8085/api/social/post/list?postType=1"      # 按类型过滤
```

预期：code=0，list 项为 `{id,userId,nickname,avatarUrl,recipeId,content,imageUrls:[],postType,likeCount,commentCount,createdAt}`，按创建时间倒序，仅 status=1 的帖子。

### 5.3 GET /api/social/post/{id} —— 帖子详情（公开，契约外补充接口）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/post/$postId" | ConvertTo-Json -Depth 5
```

预期：code=0，data 同列表项结构；不存在/已删除 → 40400`帖子不存在`。

### 5.4 POST /api/social/comment —— 发评论（鉴权）

```powershell
# 评论帖子（targetType=2，会使帖子 commentCount+1）
$c = Invoke-RestMethod -Uri "http://localhost:8085/api/social/comment" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{targetType=2;targetId=$postId;content="看起来太香了！"} | ConvertTo-Json)
$commentId = $c.data

# 评论菜谱某一步（targetType=1，stepId 可选，针对步骤踩坑反馈）
Invoke-RestMethod -Uri "http://localhost:8085/api/social/comment" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{targetType=1;targetId=1;stepId=5;content="滑炒鸡丁这步油温很关键"} | ConvertTo-Json)

# 楼中楼回复（parentId 指向父评论）
Invoke-RestMethod -Uri "http://localhost:8085/api/social/comment" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{targetType=2;targetId=$postId;content="+1 馋了";parentId=$commentId} | ConvertTo-Json)
```

预期：code=0，data 为**新评论ID**（数字）；评论帖子后再查 5.3，`commentCount` +1。
约束：targetType 必填 1(菜谱)/2(帖子)；targetId 必填；content 必填 ≤500 字；stepId/parentId 可空。

### 5.5 GET /api/social/comment/list —— 评论列表（公开）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/comment/list?targetType=2&targetId=$postId&page=1&size=10" | ConvertTo-Json -Depth 5
```

预期：code=0，list 项为 `{id,userId,nickname,avatarUrl,stepId,content,parentId,likeCount,createdAt}`，按创建时间倒序。

### 5.6 POST /api/social/like —— 点赞 toggle（鉴权）

```powershell
# 第一次：点赞
Invoke-RestMethod -Uri "http://localhost:8085/api/social/like" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{targetType=1;targetId=$postId} | ConvertTo-Json)
# 预期 data: {"liked":true,"likeCount":1}

# 第二次（同一对象）：取消点赞
Invoke-RestMethod -Uri "http://localhost:8085/api/social/like" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{targetType=1;targetId=$postId} | ConvertTo-Json)
# 预期 data: {"liked":false,"likeCount":0}

# 点赞评论（targetType=2）/ 点赞菜谱（targetType=3，仅记流水，计数按记录表统计）
Invoke-RestMethod -Uri "http://localhost:8085/api/social/like" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{targetType=2;targetId=$commentId} | ConvertTo-Json)
Invoke-RestMethod -Uri "http://localhost:8085/api/social/like" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{targetType=3;targetId=1} | ConvertTo-Json)
```

> **注意 targetType 编号体系**：点赞为 1帖子/2评论/3菜谱（与评论的 1菜谱/2帖子不同，勿混用）。帖子/评论点赞同步维护自身表 like_count（不小于0）；菜谱点赞无本地计数列，likeCount 按 `social_like_record` 实时统计。

### 5.7 POST /api/social/rating —— 菜谱评分（鉴权，重复评分更新）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/rating" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{recipeId=1;score=5} | ConvertTo-Json)
# 预期 code=0, data=null

# 同一用户重复评分 → 更新原记录（不新增行）
Invoke-RestMethod -Uri "http://localhost:8085/api/social/rating" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{recipeId=1;score=3} | ConvertTo-Json)
```

### 5.8 GET /api/social/rating/{recipeId} —— 评分统计（公开）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/rating/1"
# 预期 data: {"avgScore":3.0,"ratingCount":1}（重复评分后人数仍为1；平均分保留1位小数；无评分时 {"avgScore":0.0,"ratingCount":0}）
```

### 5.9 POST /api/social/checkin —— 打卡（鉴权）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/checkin" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{recipeId=1;note="今天做了宫保鸡丁"} | ConvertTo-Json) | ConvertTo-Json -Depth 5
```

预期（首次）：code=0，data=`{"continuousDays":1,"newBadges":[{"badgeName":"首灶之喜","icon":"🔥"}]}`。

规则：请求体字段均可空（可直接 `-Body "{}"`）；连续天数 = 昨日有打卡则昨日值+1，否则重置为 1；连续 7 天触发"七日烟火"、21 天触发"灶台常客"；**同日重复打卡 → 40900**（见 6.2）；打卡记录与徽章发放同事务。

### 5.10 GET /api/social/checkin/my?month=2026-06 —— 打卡月历（鉴权）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/checkin/my?month=2026-06" -Headers $H
# month 可省略（默认当月）。预期 data: {"dates":["2026-06-11"],"continuousDays":1}
# 注意：最近一次打卡若既非今天也非昨天，continuousDays 返回 0（连续已中断）
```

### 5.11 GET /api/social/badge/my —— 徽章墙（鉴权）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/badge/my" -Headers $H | ConvertTo-Json -Depth 4
```

预期：code=0，data 为**全部 6 枚徽章**（含未获得的），项结构 `{id,badgeName,badgeDesc,icon,obtained,obtainedAt}`；已获得 `obtained=true` 且带 `obtainedAt`，未获得 `obtained=false`、`obtainedAt=null`。

### 5.12 POST /api/social/share-card —— 分享卡数据（鉴权）

```powershell
Invoke-RestMethod -Uri "http://localhost:8085/api/social/share-card" -Method Post -Headers $H -ContentType "application/json;charset=utf-8" -Body (@{recipeId=1;recipeName="宫保鸡丁";caloriesKcal=350} | ConvertTo-Json) | ConvertTo-Json
```

预期（带菜谱与热量）：

```json
{
  "title": "我做了「宫保鸡丁」", "subtitle": "食研社 · 用心做好每一餐",
  "calorieText": "本餐热量约350千卡", "equivalentText": "相当于慢跑30分钟",
  "continuousDays": 1, "nickname": "社交测试员", "date": "2026-06-11"
}
```

> equivalentText 算法：`分钟 = round(热量 / 11.6)`，350/11.6≈30.2→30。空请求体（`-Body "{}"` 或不传 body）：title=`今日烹饪打卡`、subtitle=`食研社 · 健康饮食每一天`，calorieText/equivalentText 为 null。

### 5.13 全流程串测脚本（发帖→点赞→评论→打卡→查徽章→分享卡）

保存为 `social-e2e.ps1` 直接执行（需 8081/8085 已启动）：

```powershell
$ErrorActionPreference = "Stop"
$base = "http://localhost:8085/api/social"
$ct = "application/json;charset=utf-8"

# 0) 登录拿 Token（首次自动注册）
try {
  $auth = Invoke-RestMethod -Uri "http://localhost:8081/api/user/login" -Method Post -ContentType $ct -Body (@{username="social_tester";password="Test@1234"} | ConvertTo-Json)
} catch {
  $auth = Invoke-RestMethod -Uri "http://localhost:8081/api/user/register" -Method Post -ContentType $ct -Body (@{username="social_tester";password="Test@1234";nickname="社交测试员"} | ConvertTo-Json)
}
$H = @{ Authorization = "Bearer $($auth.data.token)" }
Write-Host "0) Token 获取成功"

# 1) 发帖（首帖应触发"晒盘新秀"徽章）
$post = Invoke-RestMethod -Uri "$base/post" -Method Post -Headers $H -ContentType $ct -Body (@{recipeId=1;content="宫保鸡丁大成功！";imageUrls=@("https://picsum.photos/seed/e2e/600/400");postType=1} | ConvertTo-Json)
$postId = $post.data.id
Write-Host ("1) 发帖成功 postId={0}, 新徽章: {1}" -f $postId, (($post.data.newBadges | ForEach-Object badgeName) -join ","))

# 2) 点赞帖子
$like = Invoke-RestMethod -Uri "$base/like" -Method Post -Headers $H -ContentType $ct -Body (@{targetType=1;targetId=$postId} | ConvertTo-Json)
Write-Host ("2) 点赞: liked={0}, likeCount={1}" -f $like.data.liked, $like.data.likeCount)

# 3) 评论帖子 + 查列表（评论数应为1，昵称应为"社交测试员"）
$c = Invoke-RestMethod -Uri "$base/comment" -Method Post -Headers $H -ContentType $ct -Body (@{targetType=2;targetId=$postId;content="自评一条，留香"} | ConvertTo-Json)
$cl = Invoke-RestMethod -Uri "$base/comment/list?targetType=2&targetId=$postId"
Write-Host ("3) 评论 id={0}; 列表共{1}条, 首条昵称={2}" -f $c.data, $cl.data.total, $cl.data.list[0].nickname)

# 4) 打卡（首卡应触发"首灶之喜"徽章；同日重复打卡应 40900）
$ck = Invoke-RestMethod -Uri "$base/checkin" -Method Post -Headers $H -ContentType $ct -Body (@{recipeId=1;note="打卡"} | ConvertTo-Json)
Write-Host ("4) 打卡: 连续{0}天, 新徽章: {1}" -f $ck.data.continuousDays, (($ck.data.newBadges | ForEach-Object badgeName) -join ","))
$dup = Invoke-RestMethod -Uri "$base/checkin" -Method Post -Headers $H -ContentType $ct -Body "{}"
Write-Host ("4x) 重复打卡: code={0}, message={1}（预期 40900 今日已打卡）" -f $dup.code, $dup.message)

# 5) 评分两次（重复评分应更新而非新增）
Invoke-RestMethod -Uri "$base/rating" -Method Post -Headers $H -ContentType $ct -Body (@{recipeId=1;score=5} | ConvertTo-Json) | Out-Null
Invoke-RestMethod -Uri "$base/rating" -Method Post -Headers $H -ContentType $ct -Body (@{recipeId=1;score=4} | ConvertTo-Json) | Out-Null
$r = Invoke-RestMethod -Uri "$base/rating/1"
Write-Host ("5) 评分统计: avg={0}, count={1}（重复评分后 count 不增）" -f $r.data.avgScore, $r.data.ratingCount)

# 6) 查徽章墙（应至少 2 枚 obtained=true：晒盘新秀 + 首灶之喜）
$b = Invoke-RestMethod -Uri "$base/badge/my" -Headers $H
$got = ($b.data | Where-Object { $_.obtained }) | ForEach-Object badgeName
Write-Host ("6) 徽章墙: 共{0}枚, 已获得: {1}" -f $b.data.Count, ($got -join ","))

# 7) 分享卡
$card = Invoke-RestMethod -Uri "$base/share-card" -Method Post -Headers $H -ContentType $ct -Body (@{recipeName="宫保鸡丁";caloriesKcal=350} | ConvertTo-Json)
Write-Host ("7) 分享卡: {0} | {1} | {2} | 连续{3}天 | {4}" -f $card.data.title, $card.data.calorieText, $card.data.equivalentText, $card.data.continuousDays, $card.data.nickname)
```

> 说明：业务失败（40900/40400/40000）HTTP 状态仍为 200，`Invoke-RestMethod` 不抛异常，直接读返回对象的 `code` 即可（脚本第 4x 步即此写法）；仅鉴权失败为 HTTP 401（需 try/catch 读 `$_.ErrorDetails.Message`）。

## 6. 异常场景用例（以实际代码逻辑为准）

| # | 场景 | 操作 | 预期 |
|---|---|---|---|
| 6.1 | 未登录写操作 | 发帖/评论/点赞/评分/打卡/徽章/分享卡不带 Token | HTTP **401**，`{"code":40100,"message":"未登录或登录已过期"}` |
| 6.2 | **同日重复打卡** | 当天第二次 POST `/checkin` | **40900**`今日已打卡`；日志 WARN `拦截重复打卡` |
| 6.3 | **重复评分更新** | 同一用户对同一菜谱先 5 星后 3 星 | 两次均 code=0；`ratingCount` 不变、`avgScore` 变 3.0；`social_rating` 仅一行；日志 `重复评分, 原评分5星 -> 新评分3星` |
| 6.4 | 评论不存在的帖子 | POST `/comment` body `{"targetType":2,"targetId":99999,...}` | **40400**`评论的帖子不存在`（targetType=1 菜谱**不校验**存在性，直接成功） |
| 6.5 | 点赞不存在的对象 | like targetType=1/2 + 不存在 ID | **40400**`点赞的帖子不存在` / `点赞的评论不存在`（targetType=3 菜谱不校验，记流水成功） |

## 9. 已知差异（代码 vs 契约）

1. **新增接口**：代码额外提供 `GET /api/social/post/{id}`（帖子详情，公开），契约第 5 节未列出（共 11 个契约接口 + 1 个补充接口 = 12 个）。
2. **发帖响应**：契约未定义发帖响应结构；代码返回 `data: {id, newBadges:[{badgeName,icon}]}`（发帖也会触发 POST_COUNT 类徽章并随响应返回）。发评论响应 data 为新评论 ID（数字），契约同样未定义。
3. **分享卡等效文案取整方式**：契约写"热量/11.6 分钟**取整**"；代码用 `Math.round` **四舍五入**（如 250 千卡 → 21.55 → 22 分钟，按向下取整应为 21，存在 ±1 分钟差异可能）。
4. **targetType 编号体系不统一（易混点，非冲突）**：评论 targetType 为 1菜谱/2帖子，点赞 targetType 为 1帖子/2评论/3菜谱（与 sql/05_social.sql 注释一致，契约未枚举取值）。
5. **recipeId 不做跨服务校验**：评论菜谱、评分、打卡、点赞菜谱时不调用菜谱服务校验 recipeId 是否存在（传不存在的 ID 也会成功），契约未要求校验，特此说明。
6. **同日重复打卡 40900**、打卡连续中断归零等为代码实现的业务细则，契约仅写"计算连续天数并触发徽章"，行为方向一致。
