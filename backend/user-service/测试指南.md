# 用户与画像服务（user-service）测试指南

## 一、模块简介

- **职责**：账号注册/登录（JWT 签发方）、用户公开信息、个人主页、健康档案、Z世代标签、饮食日历。
- **端口**：`8081`，接口前缀 `/api/user`。
- **表前缀**：`user_`，共 4 张表：`user_account`（账号）、`user_health_profile`（健康档案）、`user_tag`（画像标签）、`user_diet_record`（饮食记录）。
- **鉴权白名单**：`/api/user/register`、`/api/user/login`、`/api/user/public/**`，其余接口必须带 `Authorization: Bearer <token>`。
- **统一响应体**：`{"code":0,"message":"...","data":...,"timestamp":"..."}`；code=0 成功。

## 二、环境准备

| 环境 | 数据库 | 说明 |
|---|---|---|
| dev（默认） | H2 内存库（MODE=MySQL） | **零外部依赖**：JPA 自动建表（`ddl-auto: update`）+ 启动自动创建演示账号 `demo/123456`（含档案与 2 个标签） |
| uat | MySQL 8.0 `localhost:3306/food_app` | 需先执行 `sql/00_database.sql` 与 `sql/01_user.sql` 建库建表（`ddl-auto: none`）；账号密码用环境变量 `DB_USERNAME`/`DB_PASSWORD`（默认 root/root），JWT 密钥用 `JWT_SECRET` |
| prod | MySQL 8.0（同 uat） | 另外关闭 SQL 打印、根日志级别 INFO |

切换 profile（默认 dev，无需任何参数）：

```powershell
# 方式一：命令行参数
mvn -pl user-service spring-boot:run "-Dspring-boot.run.profiles=uat"
# 方式二：环境变量
$env:SPRING_PROFILES_ACTIVE = "uat"; mvn -pl user-service spring-boot:run
```

## 三、启动方式与成功标志

```powershell
cd d:\self\food-app\backend
# 首次启动前需安装父 POM 与 common 到本地仓库（仅一次）：
mvn -N install; mvn -pl common install -DskipTests
# 启动（或直接用 -am 自动连带构建 common）：
mvn -pl user-service -am spring-boot:run
```

**启动成功标志**（约 10~20 秒，看控制台或 `logs/user-service.log`）：

- `Tomcat started on port 8081`
- `Started UserServiceApplication in X seconds`
- dev 首次启动：`[种子数据] 演示账号已创建: username=demo, userId=1, 档案与标签已就绪`（重启内存库重建，会再次出现）
- `[配置] 注册JWT鉴权拦截器，白名单：/api/user/register、/api/user/login、/api/user/public/**`

> **PowerShell 中文请求体注意（实测）**：Windows 控制台默认 GBK 代码页，直接在 `-d` 里写中文会以 GBK 字节发出，服务端报 `{"code":50000}`（JSON parse error: Invalid UTF-8）。两种解决办法：
> ① 测试前先执行 `chcp 65001` 切到 UTF-8 代码页；
> ② 把请求体保存为 **UTF-8 编码文件**，用 `curl.exe --data-binary "@body.json"` 发送（最稳妥）。
> 以下示例统一使用 `curl.exe`（PowerShell 里 `curl` 是 `Invoke-WebRequest` 别名，参数不同）。

## 四、接口测试用例

### 接口总览（共 11 个，与代码 Controller 一一对应）

| # | 方法 | 路径 | 鉴权 | 说明 | 成功 code/message |
|---|---|---|---|---|---|
| 1 | POST | /api/user/register | 否 | 注册（自动建空健康档案并签发 token） | 0 / 注册成功 |
| 2 | POST | /api/user/login | 否 | 登录 | 0 / 登录成功 |
| 3 | GET | /api/user/public/{id} | 否 | 公开信息（供社交服务调用） | 0 / 操作成功 |
| 4 | GET | /api/user/profile | 是 | 个人主页（用户+档案+标签） | 0 / 操作成功 |
| 5 | PUT | /api/user/profile | 是 | 更新健康档案（字段全可选） | 0 / 健康档案更新成功 |
| 6 | GET | /api/user/tag/list | 是 | 我的标签 | 0 / 操作成功 |
| 7 | POST | /api/user/tag | 是 | 添加标签 | 0 / 标签添加成功 |
| 8 | DELETE | /api/user/tag/{id} | 是 | 删除标签（仅本人） | 0 / 标签删除成功 |
| 9 | POST | /api/user/diet | 是 | 记录饮食 | 0 / 饮食记录成功 |
| 10 | GET | /api/user/diet/daily?date= | 是 | 当日统计 | 0 / 操作成功 |
| 11 | GET | /api/user/diet/month?month= | 是 | 月度日历 | 0 / 操作成功 |

> 以下 `<TOKEN>` 替换为注册/登录响应的 `data.token`；演示账号 `demo/123456` 可直接登录获取。

### 1. 注册 POST /api/user/register

```powershell
curl.exe -s -X POST "http://localhost:8081/api/user/register" -H "Content-Type: application/json" -d '{\"username\":\"xiaoyan\",\"password\":\"abc123456\",\"nickname\":\"xiaoyan\"}'
```

预期响应（token 为动态值；昵称含中文请用 UTF-8 文件方式发送）：

```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzM4NCJ9....",
    "user": { "id": 2, "username": "xiaoyan", "nickname": "xiaoyan", "avatarUrl": null }
  },
  "timestamp": "2026-06-11T19:49:52"
}
```

入参校验：username 3~50 字符、password 6~32 字符、nickname 必填 ≤50 字符。

### 2. 登录 POST /api/user/login

```powershell
curl.exe -s -X POST "http://localhost:8081/api/user/login" -H "Content-Type: application/json" -d '{\"username\":\"demo\",\"password\":\"123456\"}'
```

预期：`code=0, message=登录成功`，data 结构同注册。
PowerShell 一行拿 token 备用：

```powershell
$token = (Invoke-RestMethod -Method Post -Uri "http://localhost:8081/api/user/login" -ContentType "application/json" -Body '{"username":"demo","password":"123456"}').data.token
```

### 3. 公开信息 GET /api/user/public/{id}

```powershell
curl.exe -s "http://localhost:8081/api/user/public/1"
```

预期：`{"code":0,"data":{"id":1,"nickname":"美食家小研","avatarUrl":null},...}`；id 不存在返回 `40400 用户不存在`。

### 4. 个人主页 GET /api/user/profile

```powershell
curl.exe -s "http://localhost:8081/api/user/profile" -H "Authorization: Bearer <TOKEN>"
```

预期响应（demo 账号，新注册账号 profile 各字段为 null、tags 为 []）：

```json
{
  "code": 0,
  "data": {
    "user": { "id": 1, "username": "demo", "nickname": "美食家小研", "avatarUrl": null, "gender": 0, "birthDate": null },
    "profile": { "heightCm": 170.0, "weightKg": 65.0, "allergyHistory": "海鲜", "dietPreference": null, "healthGoal": "减脂", "dailyCalorieTarget": 1800 },
    "tags": [ { "id": 1, "tagName": "减脂期", "tagType": 1 }, { "id": 2, "tagName": "健身后", "tagType": 2 } ]
  }
}
```

### 5. 更新健康档案 PUT /api/user/profile

```powershell
curl.exe -s -X PUT "http://localhost:8081/api/user/profile" -H "Authorization: Bearer <TOKEN>" -H "Content-Type: application/json" -d '{\"heightCm\":171.5,\"weightKg\":63.2,\"dailyCalorieTarget\":1700}'
```

预期：`{"code":0,"message":"健康档案更新成功","data":null}`。全部字段可选（nickname/avatarUrl/heightCm/weightKg/allergyHistory/dietPreference/healthGoal/dailyCalorieTarget），只更新传入字段；再调接口 4 验证生效。

### 6. 我的标签 GET /api/user/tag/list

```powershell
curl.exe -s "http://localhost:8081/api/user/tag/list" -H "Authorization: Bearer <TOKEN>"
```

预期：`{"code":0,"data":[{"id":1,"tagName":"减脂期","tagType":1},...]}`

### 7. 添加标签 POST /api/user/tag

```powershell
# 中文标签名先 chcp 65001，或用 UTF-8 文件体
curl.exe -s -X POST "http://localhost:8081/api/user/tag" -H "Authorization: Bearer <TOKEN>" -H "Content-Type: application/json" -d '{\"tagName\":\"aoyedang\",\"tagType\":1}'
```

预期：`{"code":0,"message":"标签添加成功","data":{"id":3,"tagName":"...","tagType":1}}`。tagType 仅 1（人群）或 2（状态）。

### 8. 删除标签 DELETE /api/user/tag/{id}

```powershell
curl.exe -s -X DELETE "http://localhost:8081/api/user/tag/3" -H "Authorization: Bearer <TOKEN>"
```

预期：`{"code":0,"message":"标签删除成功","data":null}`；删除不存在/他人标签返回 `40400 标签不存在`。

### 9. 记录饮食 POST /api/user/diet

```powershell
curl.exe -s -X POST "http://localhost:8081/api/user/diet" -H "Authorization: Bearer <TOKEN>" -H "Content-Type: application/json" -d '{\"recordDate\":\"2026-06-11\",\"mealType\":1,\"recipeId\":1,\"recipeName\":\"gongbaojiding\",\"caloriesKcal\":450.5,\"carbsG\":60.2,\"proteinG\":18.5,\"fatG\":12.3}'
```

预期：`{"code":0,"message":"饮食记录成功","data":null}`。mealType：1 早餐 2 午餐 3 晚餐 4 加餐；recipeId/三大营养素可空。

### 10. 当日统计 GET /api/user/diet/daily?date=yyyy-MM-dd

```powershell
curl.exe -s "http://localhost:8081/api/user/diet/daily?date=2026-06-11" -H "Authorization: Bearer <TOKEN>"
```

预期：

```json
{
  "code": 0,
  "data": {
    "records": [ { "id": 1, "mealType": 1, "recipeName": "...", "caloriesKcal": 450.50, "carbsG": 60.20, "proteinG": 18.50, "fatG": 12.30 } ],
    "totalCalories": 450.50, "totalCarbs": 60.20, "totalProtein": 18.50, "totalFat": 12.30,
    "calorieTarget": 1800, "exceedTarget": false
  }
}
```

超标逻辑：再记录一条 `caloriesKcal=1500` 后重查，`exceedTarget` 变 `true`，同时日志输出「当日摄入超标」。未设置热量目标时 `calorieTarget=null、exceedTarget=false`。

### 11. 月度日历 GET /api/user/diet/month?month=yyyy-MM

```powershell
curl.exe -s "http://localhost:8081/api/user/diet/month?month=2026-06" -H "Authorization: Bearer <TOKEN>"
```

预期（仅返回有记录的日期）：`{"code":0,"data":[{"date":"2026-06-11","totalCalories":450.50}]}`

## 五、异常场景用例（以实际代码为准）

| 场景 | 操作 | 预期 code | 预期 message |
|---|---|---|---|
| 重复注册 | 用已存在 username 注册 | 40900 | 用户名已存在 |
| 密码错误 / 用户名不存在 | 登录（两种错误统一提示，防账号枚举） | 40000 | 用户名或密码错误 |
| 账号被禁用（status=0） | 登录 | 40300 | 账号已被禁用，请联系管理员 |
| 未带 token | GET /api/user/profile（HTTP 401） | 40100 | 未登录或登录已过期 |
| token 过期/伪造 | 任意鉴权接口（HTTP 401） | 40100 | 未登录或登录已过期 |
| 参数校验失败 | 注册 username 仅 2 字符 | 40000 | username 用户名长度需在3~50个字符之间 |
| 餐次越界 | POST /diet mealType=9 | 40000 | mealType 餐次取值范围为1~4 |
| 日期格式非法 | GET /diet/daily?date=2026/06/11 | 40000 | 日期格式应为yyyy-MM-dd |
| 月份格式非法 | GET /diet/month?month=202606 | 40000 | 月份格式应为yyyy-MM |
| 重复添加同名标签 | POST /tag 同名 | 40900 | 标签已存在 |
| 删除他人/不存在标签 | DELETE /tag/{id} | 40400 | 标签不存在 |
| 查询不存在用户 | GET /public/999 | 40400 | 用户不存在 |
| 请求体非 UTF-8 中文 | GBK 编码 JSON（HTTP 200） | 50000 | 服务内部错误，请稍后重试 |

示例（实测验证过）：

```powershell
# 重复注册 → 40900
curl.exe -s -X POST "http://localhost:8081/api/user/register" -H "Content-Type: application/json" -d '{\"username\":\"demo\",\"password\":\"abc123456\",\"nickname\":\"dup\"}'
# 无 token → 40100（HTTP 401）
curl.exe -s "http://localhost:8081/api/user/profile"
# 参数校验 → 40000
curl.exe -s -X POST "http://localhost:8081/api/user/register" -H "Content-Type: application/json" -d '{\"username\":\"ab\",\"password\":\"123\",\"nickname\":\"x\"}'
```

## 七、数据库验证点

**dev（H2 控制台）**：浏览器访问 `http://localhost:8081/h2-console`，JDBC URL 填 `jdbc:h2:mem:userdb`，用户名 `sa`，密码留空。

```sql
-- 账号表：密码必须是 BCrypt 密文（$2a$ 开头），绝无明文
SELECT id, username, nickname, LEFT(password, 7) AS pwd_prefix, status FROM user_account;
-- 注册后自动建档：每个账号应有一条档案
SELECT * FROM user_health_profile;
-- 标签归属
SELECT * FROM user_tag WHERE user_id = 1;
-- 饮食记录与当日汇总核对
SELECT record_date, SUM(calories_kcal) FROM user_diet_record WHERE user_id = 1 GROUP BY record_date;
```

**uat/prod（MySQL）**：`mysql -uroot -p food_app` 后执行同样 SQL（表结构见 `sql/01_user.sql`，表与字段带中文 COMMENT）。

验证要点：

1. 注册成功后 `user_account` 新增一行且 `password` 为 BCrypt 60 位密文；`user_health_profile` 同步新增一行（全空字段）。
2. PUT /profile 后对应字段更新，未传字段保持原值。
3. 删除标签后 `user_tag` 行被物理删除。
4. H2 为内存库，**重启即清空**，种子数据会重新初始化。
