## 三、统一技术约定

### 3.1 后端（Java 模块）
- **Spring Boot 3.3.5 + Java 21 + Spring Data JPA**，包名 `com.foodapp.<模块名>`
- 启动类必须 `scanBasePackages = {"com.foodapp.common", "com.foodapp.<模块>"}`，并 `@EnableJpaRepositories` / `@EntityScan` 指向自身包
- **每个 public 方法必须有 Javadoc 中文说明；关键判断逻辑（鉴权失败、状态流转、替换计算、库存判断等）必须有 log.info / log.warn 日志输出**
- 统一响应体 `Result<T>`（common 提供），统一异常 `BusinessException` + 全局异常处理器（common 提供）
- 鉴权：common 提供 `JwtUtil` 与 `AuthInterceptor`，各服务注册拦截器并配置白名单
- 日志：common 提供 `AccessLogFilter` 对所有请求/响应做日志化输出；logback 同时输出控制台与 `logs/<服务名>.log` 滚动文件

### 3.2 多环境
- `application.yml` 中 `spring.profiles.active: dev`（**默认 dev**）
- `application-dev.yml`：H2 内存库（MODE=MySQL），`ddl-auto: update`，启动自动执行 `data.sql` 种子数据
- `application-uat.yml` / `application-prod.yml`：MySQL 8.0（`jdbc:mysql://localhost:3306/food_app`），prod 日志级别 INFO、关闭 SQL 打印
- JWT 密钥、跨服务地址均按环境分别配置

### 3.3 数据库
- 单库 `food_app`，**不同模块表用前缀区分**；表与字段一律带中文 `COMMENT`
- MySQL 正式 DDL 统一放 `sql/` 目录（01~06 按模块编号）
- JPA 实体使用 `@Comment` 注解保证 dev 自动建表也带中文说明

### 3.4 安全
- 密码 BCrypt 加密存储；JWT HS256 签名（密钥按环境区分）
- 入参 `jakarta.validation` 校验；ORM 参数化查询防 SQL 注入
- CORS 仅放行前端来源；敏感字段（密码）不出现在任何响应与日志中

### 3.5 前端
- React 18 + Vite 5 + Tailwind CSS 3 + Zustand + React Router 6
- **移动优先，PC 端响应式兼容**（max-w 居中、断点布局）
- 视觉：新中式美学 —— 宣纸米白底、朱砂暖橙主色、黛绿辅助、墨色文字，详见 frontend 内设计约定
- Vite 代理：`/api/user→8081`、`/api/recipe→8082`、`/api/kitchen→8083`、`/api/search→8084`、`/api/social→8085`、`/api/ai→8086`、`/api/file→8087`

### 3.6 接口路径约定
各服务上下文路径统一为 `/api/<模块>`，例如用户服务全部接口以 `/api/user/**` 开头，便于网关/代理按前缀路由。

## 四、测试文档约定
每个模块根目录必须有 `测试指南.md`：包含启动方式、环境说明、每个接口的 curl 示例与预期结果、异常场景用例。

## 五、启动顺序（dev）
1. 后端：`cd backend && mvn -pl <service> spring-boot:run`（任意顺序；菜谱服务建议先启动，搜索/厨房依赖其接口）
2. AI 服务：`cd backend/ai-service && pip install -r requirements.txt && python main.py`
3. 前端：`cd frontend && npm install && npm run dev`
