# 接口契约（所有模块开发必须严格遵守）

> 统一响应体：`{"code":0,"message":"操作成功","data":...,"timestamp":"..."}`，code=0 成功；40100 未登录；40000 参数错误；40900 业务冲突；50000 服务错误。
> 鉴权接口需请求头 `Authorization: Bearer <token>`。
> 分页响应统一为 `{"total":100,"page":1,"size":10,"list":[...]}`。
> 所有 JSON 字段一律小驼峰命名。

## 1. 用户服务 user-service :8081（前缀 /api/user）

| 方法 | 路径 | 鉴权 | 说明 |
|---|---|---|---|
| POST | /api/user/register | 否 | 注册。req: `{username,password,nickname}`；resp data: `{token,user:{id,username,nickname,avatarUrl}}` |
| POST | /api/user/login | 否 | 登录。req: `{username,password}`；resp 同上 |
| GET | /api/user/public/{id} | 否 | 公开信息（供社交服务/前端展示）。resp data: `{id,nickname,avatarUrl}` |
| GET | /api/user/profile | 是 | 个人主页。resp data: `{user:{id,username,nickname,avatarUrl,gender,birthDate},profile:{heightCm,weightKg,allergyHistory,dietPreference,healthGoal,dailyCalorieTarget},tags:[{id,tagName,tagType}]}` |
| PUT | /api/user/profile | 是 | 更新健康档案。req: `{nickname?,avatarUrl?,heightCm?,weightKg?,allergyHistory?,dietPreference?,healthGoal?,dailyCalorieTarget?}` |
| GET | /api/user/tag/list | 是 | 我的标签。resp data: `[{id,tagName,tagType}]` |
| POST | /api/user/tag | 是 | 添加标签。req: `{tagName,tagType}` |
| DELETE | /api/user/tag/{id} | 是 | 删除标签 |
| POST | /api/user/diet | 是 | 记录饮食。req: `{recordDate:"2026-06-11",mealType:1,recipeId?,recipeName,caloriesKcal,carbsG,proteinG,fatG}` |
| GET | /api/user/diet/daily?date=2026-06-11 | 是 | 当日统计。resp data: `{records:[{id,mealType,recipeName,caloriesKcal,carbsG,proteinG,fatG}],totalCalories,totalCarbs,totalProtein,totalFat,calorieTarget,exceedTarget}` |
| GET | /api/user/diet/month?month=2026-06 | 是 | 月度日历。resp data: `[{date:"2026-06-01",totalCalories}]` |

白名单：`/api/user/register`、`/api/user/login`、`/api/user/public/**`

## 2. 菜谱服务 recipe-service :8082（前缀 /api/recipe）

**RecipeSummary**（列表项）：`{id,title,coverUrl,cuisineType,cuisineName,difficulty,totalTimeMin,caloriesKcal,carbsG,proteinG,fatG,description,viewCount,likeCount,tags:[string]}`

**Step**：`{id,recipeId,phase,stepIndex,actionTitle,detail,mediaUrl,timerSec,firePower}`（phase ∈ PREPARE/WASH/CUT/COOK/PLATE）

| 方法 | 路径 | 鉴权 | 说明 |
|---|---|---|---|
| GET | /api/recipe/list | 否 | 分页查询。query: `page=1&size=10&cuisineType?&difficulty?&keyword?&maxCalories?&tagName?&ingredients?`（ingredients 逗号分隔食材名，匹配含任一食材的菜谱=冰箱清理模式）。resp: 分页结构，list 为 RecipeSummary |
| GET | /api/recipe/{id} | 否 | 详情。resp data: `{info:RecipeSummary&{tips,servings,sourceType},stepsByPhase:{PREPARE:[Step],WASH:[...],CUT:[...],COOK:[...],PLATE:[...]},ingredients:[{id,ingredientId,ingredientName,amount,unit,isEssential}],suitableTags:[string],unsuitableTags:[string],phaseWeights:{PREPARE:10,WASH:5,CUT:15,COOK:60,PLATE:10}}` |
| GET | /api/recipe/{id}/steps | 否 | 步骤平铺（厨房服务调用）。resp data: `[Step]`（按 phase 顺序 PREPARE→PLATE、再按 stepIndex 排序） |
| POST | /api/recipe/{id}/substitute | 否 | 智能替换。req: `{scene}`（减脂/控糖/素食/低嘌呤）。resp data: `{originalCalories,newCalories,calorieDiff,substitutions:[{sourceName,targetName,reason,calorieDiff}]}` |
| POST | /api/recipe/{id}/view | 否 | 浏览数+1 |
| POST | /api/recipe/ugc | 是 | UGC上传（五步法模板，必须含全部5阶段）。req: `{title,coverUrl?,cuisineType,difficulty,totalTimeMin,servings,description?,tips?,ingredients:[{ingredientId,amount,unit,isEssential}],steps:[{phase,stepIndex,actionTitle,detail,mediaUrl?,timerSec?,firePower?}],suitableTagIds:[],unsuitableTagIds:[]}`；resp data: `{id}`（status=2待审核） |
| GET | /api/recipe/ingredient/list | 否 | 食材库。resp data: `[{id,name,category,caloriesPer100g,carbsPer100g,proteinPer100g,fatPer100g,tabooNote}]` |
| GET | /api/recipe/tag/list | 否 | 标签字典。resp data: `[{id,tagName,tagType}]` |

白名单：除 `/api/recipe/ugc` 外全部公开。

## 3. 厨房服务 kitchen-service :8083（前缀 /api/kitchen，全部需鉴权）

**SessionVO**：`{id,recipeId,recipeName,currentPhase,currentStepIndex,status,progressPercent,currentStep:Step,steps:[Step],timers:[TimerVO],startedAt,finishedAt}`
**TimerVO**：`{id,timerName,totalSec,remainSec,status}`
进度算法：阶段权重 PREPARE10/WASH5/CUT15/COOK60/PLATE10，阶段内按步骤序号线性折算。

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | /api/kitchen/session | 开始烹饪。req: `{recipeId}`（调菜谱服务取步骤）。resp data: SessionVO |
| GET | /api/kitchen/session/{id} | 查询会话。resp data: SessionVO |
| POST | /api/kitchen/session/{id}/next | 下一步（阶段末自动进入下一阶段；PLATE 最后一步→自动完成）。resp data: SessionVO |
| POST | /api/kitchen/session/{id}/prev | 上一步（仅允许阶段内回退，跨阶段回退返回 40900）。resp data: SessionVO |
| POST | /api/kitchen/session/{id}/finish | 主动完成。resp data: SessionVO |
| POST | /api/kitchen/session/{id}/abandon | 放弃烹饪 |
| POST | /api/kitchen/session/{id}/timer | 开计时器（支持多个并行）。req: `{timerName,totalSec}`。resp data: TimerVO |
| GET | /api/kitchen/session/{id}/timer | 计时器列表（到点自动置为已完成）。resp data: `[TimerVO]` |
| POST | /api/kitchen/timer/{timerId}/cancel | 取消计时器 |
| POST | /api/kitchen/session/{id}/voice | 语音指令。req: `{commandText}`。解析规则：含"下一步/继续"→NEXT_STEP；"上一步/回退"→PREV_STEP；"还有多久/几分钟"→QUERY_TIMER；"计时"→START_TIMER；其余 UNKNOWN。resp data: `{parsedAction,message,session:SessionVO?}` |

## 4. 搜索服务 search-service :8084（前缀 /api/search）

| 方法 | 路径 | 鉴权 | 说明 |
|---|---|---|---|
| GET | /api/search/recipes | 否(可带token) | 搜索。query 同 /api/recipe/list 另加 `searchType`(1关键词2食材3营养4人群)。透传调菜谱服务；keyword 非空时热搜计数+1；带 token 时写搜索历史。resp 同菜谱分页 |
| GET | /api/search/hot | 否 | 热搜 TOP10。resp data: `[{keyword,searchCount}]` |
| GET | /api/search/history | 是 | 最近20条。resp data: `[{id,keyword,searchType,createdAt}]` |
| DELETE | /api/search/history | 是 | 清空历史 |
| GET | /api/search/recommend?limit=10 | 否(可带token) | 个性化推荐：有 token→调用户服务取标签映射菜谱标签聚合；游客→按热度。resp data: `{reason:"推荐理由文案",list:[RecipeSummary]}` |

## 5. 社交服务 social-service :8085（前缀 /api/social）

评论/帖子展示时需补充用户昵称头像：调用户服务 `/api/user/public/{id}`（内存缓存10分钟）。

| 方法 | 路径 | 鉴权 | 说明 |
|---|---|---|---|
| POST | /api/social/comment | 是 | 发评论。req: `{targetType,targetId,stepId?,content,parentId?}` |
| GET | /api/social/comment/list?targetType=&targetId=&page=&size= | 否 | 评论列表。resp list 项: `{id,userId,nickname,avatarUrl,stepId,content,parentId,likeCount,createdAt}` |
| POST | /api/social/post | 是 | 发帖。req: `{recipeId?,content,imageUrls:[string],postType}` |
| GET | /api/social/post/list?postType?&page=&size= | 否 | 作品墙。resp list 项: `{id,userId,nickname,avatarUrl,recipeId,content,imageUrls:[],postType,likeCount,commentCount,createdAt}` |
| POST | /api/social/like | 是 | 点赞toggle。req: `{targetType,targetId}`。resp data: `{liked,likeCount}` |
| POST | /api/social/rating | 是 | 评分。req: `{recipeId,score}`(1-5，重复评分则更新) |
| GET | /api/social/rating/{recipeId} | 否 | resp data: `{avgScore,ratingCount}` |
| POST | /api/social/checkin | 是 | 打卡。req: `{recipeId?,note?}`。计算连续天数并触发徽章。resp data: `{continuousDays,newBadges:[{badgeName,icon}]}` |
| GET | /api/social/checkin/my?month=2026-06 | 是 | resp data: `{dates:["2026-06-01"],continuousDays}` |
| GET | /api/social/badge/my | 是 | resp data: `[{id,badgeName,badgeDesc,icon,obtained,obtainedAt}]`（全部徽章+是否已获得） |
| POST | /api/social/share-card | 是 | 分享卡数据。req: `{recipeId?,recipeName?,caloriesKcal?}`。resp data: `{title,subtitle,calorieText,equivalentText,continuousDays,nickname,date}`（equivalentText 如"相当于慢跑30分钟"=热量/11.6 分钟取整） |

## 6. AI 服务 ai-service :8086（FastAPI，前缀 /api/ai，公开）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/ai/health | 健康检查。resp: `{"code":0,"data":{"status":"UP","env":"dev"}}` |
| POST | /api/ai/substitute | req: `{"ingredients":[{"name":"五花肉","amount":200}],"scene":"减脂"}`；resp data: `{suggestions:[{sourceName,targetName,reason,calorieSavedPer100g}]}` |
| POST | /api/ai/parse-recipe | 自由文本结构化。req: `{"freeText":"..."}`；resp data: 产品需求文档中的菜谱 JSON Schema（recipe_name/steps 五阶段等），启发式拆解 + 预留大模型调用点 |

> AI 服务也用统一响应 `{"code":0,"message":"操作成功","data":...}`。
