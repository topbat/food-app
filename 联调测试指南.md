# 「食研社」端到端联调测试指南

> 适用环境：**dev**（H2 免安装，默认 profile）。执行前请按 [README.md](README.md) 第六节启动**全部服务**：
> 菜谱 8082 → 用户 8081 → 厨房 8083 / 搜索 8084 / 社交 8085 → AI 8086 →（可选）前端 5173。
>
> 统一响应体：`{"code":0,"message":"操作成功","data":...}`，每步执行后**先确认 `code` 为 0**（异常用例除外）。

## 〇、准备：会话变量与编码说明

以下命令在**同一个 PowerShell 窗口**中按顺序执行（步骤间通过变量传递 token / id）。

```powershell
# 中文显示设置（PS 5.1 控制台中文响应可能显示乱码，仅为显示问题，不影响判断 code 值；PowerShell 7+ 无此问题）
[Console]::OutputEncoding = [Text.Encoding]::UTF8

# 基础地址
$U = "http://localhost:8081"   # 用户
$R = "http://localhost:8082"   # 菜谱
$K = "http://localhost:8083"   # 厨房
$S = "http://localhost:8084"   # 搜索
$C = "http://localhost:8085"   # 社交
$A = "http://localhost:8086"   # AI
$CT = "application/json; charset=utf-8"
```

> 若 `localhost` 连接失败而服务确在运行，改用 `127.0.0.1`（IPv6 解析问题）。

## 一、注册 / 登录（用户服务 8081）

```powershell
# 1.1 注册（用户名重复时改一个，或直接走 1.2 登录）
$body = @{ username = "tester01"; password = "Test@1234"; nickname = "联调小测" } | ConvertTo-Json
$reg = Invoke-RestMethod -Uri "$U/api/user/register" -Method Post -ContentType $CT -Body $body
$reg.code   # 预期 0（用户名已存在则 40900，转 1.2）

# 1.2 登录，保存 token
$body = @{ username = "tester01"; password = "Test@1234" } | ConvertTo-Json
$login = Invoke-RestMethod -Uri "$U/api/user/login" -Method Post -ContentType $CT -Body $body
$token = $login.data.token
$uid   = $login.data.user.id
$H = @{ Authorization = "Bearer $token" }
"token=$token / uid=$uid"
```

**预期**：`code=0`，`data.token` 为 JWT 字符串，`data.user` 含 `id/username/nickname`。后续鉴权接口统一带 `-Headers $H`。

```powershell
# 1.3 （可选）查看/完善健康档案，便于后面饮食统计有热量目标
Invoke-RestMethod -Uri "$U/api/user/profile" -Method Get -Headers $H | ConvertTo-Json -Depth 6
$body = @{ heightCm = 175; weightKg = 65; healthGoal = "减脂"; dailyCalorieTarget = 1800 } | ConvertTo-Json
(Invoke-RestMethod -Uri "$U/api/user/profile" -Method Put -Headers $H -ContentType $CT -Body $body).code   # 预期 0
```

## 二、浏览菜谱列表（菜谱服务 8082）

```powershell
$list = Invoke-RestMethod -Uri "$R/api/recipe/list?page=1&size=10" -Method Get
```

**预期**：分页结构 `{total,page,size,list}`，list 项含 `id/title/difficulty/totalTimeMin/caloriesKcal` 等字段。

## 五、开始烹饪：厨房会话全流程（厨房服务 8083，需鉴权）

```powershell
# 5.1 创建会话
$body = @{ recipeId = $recipeId } | ConvertTo-Json
$sess = Invoke-RestMethod -Uri "$K/api/kitchen/session" -Method Post -Headers $H -ContentType $CT -Body $body
$sid = $sess.data.id
"sid=$sid / 当前阶段=$($sess.data.currentPhase) / 进度=$($sess.data.progressPercent)%"
```

**预期**：`code=0`，`currentPhase=PREPARE`、`currentStepIndex` 为首步、`status` 为进行中、`steps` 为菜谱全部步骤平铺。

```powershell
# 5.2 下一步 ×2（观察阶段流转与进度按权重 10/5/15/60/10 上升）
1..2 | ForEach-Object {
  $s = Invoke-RestMethod -Uri "$K/api/kitchen/session/$sid/next" -Method Post -Headers $H
  "{0}: 阶段={1} 步骤={2} 进度={3}%" -f $_, $s.data.currentPhase, $s.data.currentStepIndex, $s.data.progressPercent
}

# 5.3 上一步（阶段内回退应成功；若当前已是阶段首步则跨阶段回退 → 预期 code=40900）
$prev = Invoke-RestMethod -Uri "$K/api/kitchen/session/$sid/prev" -Method Post -Headers $H
$prev.code; $prev.message
```

```powershell
# 5.4 开计时器（支持多个并行）
$body = @{ timerName = "炖煮计时"; totalSec = 60 } | ConvertTo-Json
$t = Invoke-RestMethod -Uri "$K/api/kitchen/session/$sid/timer" -Method Post -Headers $H -ContentType $CT -Body $body
$timerId = $t.data.id
$t.data | ConvertTo-Json     # 预期 totalSec=60，remainSec≈60，status=计时中

# 查询计时器列表（remainSec 应随时间递减；到点自动置为已完成）
Invoke-RestMethod -Uri "$K/api/kitchen/session/$sid/timer" -Method Get -Headers $H | ConvertTo-Json -Depth 5

# 取消计时器
(Invoke-RestMethod -Uri "$K/api/kitchen/timer/$timerId/cancel" -Method Post -Headers $H).code   # 预期 0
```

```powershell
# 5.5 语音指令解析
foreach ($cmd in @("下一步", "上一步", "还有多久", "帮我计时", "今天天气怎么样")) {
  $body = @{ commandText = $cmd } | ConvertTo-Json
  $v = Invoke-RestMethod -Uri "$K/api/kitchen/session/$sid/voice" -Method Post -Headers $H -ContentType $CT -Body $body
  "{0} → {1}" -f $cmd, $v.data.parsedAction
}
```

**预期**：依次解析为 `NEXT_STEP / PREV_STEP / QUERY_TIMER / START_TIMER / UNKNOWN`。

## 八、发帖 + 打卡（社交服务 8085）

```powershell
# 8.1 发作品帖
$body = @{ recipeId = $recipeId; content = "第一次照着五步法做，成品居然像模像样！"; imageUrls = @("https://example.com/p1.jpg"); postType = 1 } | ConvertTo-Json
(Invoke-RestMethod -Uri "$C/api/social/post" -Method Post -Headers $H -ContentType $CT -Body $body).code   # 预期 0

# 作品墙应能看到（且补充了昵称头像，来自跨服务调用用户服务）
$wall = Invoke-RestMethod -Uri "$C/api/social/post/list?page=1&size=5" -Method Get
$wall.data.list | Select-Object nickname, content, likeCount | Format-Table

# 8.2 打卡（计算连续天数、触发徽章）
$body = @{ recipeId = $recipeId; note = "联调打卡" } | ConvertTo-Json
$ck = Invoke-RestMethod -Uri "$C/api/social/checkin" -Method Post -Headers $H -ContentType $CT -Body $body
"连续天数=$($ck.data.continuousDays) 新徽章=$($ck.data.newBadges | ConvertTo-Json -Compress)"
```

**预期**：发帖 `code=0`；作品墙 list 项含 `nickname/avatarUrl`（证明社交→用户跨服务调用正常）；首次打卡 `continuousDays=1`，`newBadges` 通常含「首次打卡」类徽章；**同日重复打卡预期 `code=40900`**。

## 十二、常见问题排查表

| 现象 | 可能原因 | 排查/解决 |
|---|---|---|
| 端口被占用，服务启动失败（`Port already in use`） | 上次进程未退出 | `Get-NetTCPConnection -LocalPort 8082 -State Listen \| Select OwningProcess` 查 PID，`Stop-Process -Id <PID> -Force` |
| `Invoke-RestMethod` 连接被拒绝 | 服务未启动 / localhost 解析为 IPv6 | 逐个访问各服务健康/列表接口确认存活；改用 `127.0.0.1` |
| 中文响应显示乱码（`æä½æå`） | PS 5.1 控制台解码问题 | 仅显示问题，`code` 判断不受影响；`[Console]::OutputEncoding=[Text.Encoding]::UTF8` 或改用 PowerShell 7 |
| 鉴权接口返回 40100 | 未带/带错 token、token 过期 | 确认 `-Headers @{Authorization="Bearer $token"}`，重新登录刷新 `$token` |
| 开始烹饪 / 搜索返回 50000 | 跨服务调用失败：菜谱服务 8082 未启动 | 先启动 recipe-service，再调厨房/搜索；看调用方 `logs/<服务名>.log` 中的远程调用错误 |
| 作品墙 nickname 为空 | 社交服务调用户服务 8081 失败 | 确认 user-service 已启动；`/api/user/public/{id}` 在白名单内可直接 GET 验证 |
| dev 重启后数据消失 | H2 为内存库，重启即清空 | 属正常现象；种子数据由 `data.sql` 重新写入，注册/会话等需重做 |
| 想直接看 dev 库数据 | — | 浏览器开 H2 控制台（各 Java 服务 `/h2-console`，JDBC URL 见各自 `application-dev.yml`，通常 `jdbc:h2:mem:...`） |
| uat/prod 启动报数据库连接失败 | MySQL 未启动 / 未建库 | 先执行 `sql/00_database.sql`~`05_social.sql`；确认账号密码与 `application-uat.yml` 一致 |
| AI 服务 8086 启动报 `ModuleNotFoundError` | 依赖未装 | `cd backend\ai-service; python -m pip install -r requirements.txt`（pip 不在 PATH 时用 `python -m pip`） |
| 请求中文参数后规则不命中 | 请求体编码不是 UTF-8 | `-ContentType "application/json; charset=utf-8"` 必须带 charset；或参考 ai-service 测试指南用 UTF-8 字节发送 |
| 前端页面接口 404/跨域 | 未走 Vite 代理或后端未起 | 通过 `http://localhost:5173` 访问（代理 `/api/*` 转各端口）；检查 `frontend/vite.config.js` 与对应后端是否存活 |
