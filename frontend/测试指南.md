# 前端 H5/PC 测试指南

> 模块：菜谱助手前端（React 18 + Vite 5 + Tailwind CSS 3 + Zustand + React Router 6），端口 **5173**。
> 移动优先、PC 端响应式兼容（max-w 居中 + 断点布局），新中式视觉（宣纸米白底、朱砂暖橙主色、黛绿辅助、墨色文字）。

## 一、环境准备

| 项 | 要求 |
|---|---|
| Node.js | ≥ 18（本机已验证 v25.8.1） |
| 包管理 | npm ≥ 9 |
| 后端依赖 | 按需启动 8081~8086 各服务（见根目录 README.md 启动顺序；只测纯页面渲染可不启动，但接口数据为空并出现错误 Toast） |

```powershell
cd d:\self\food-app\frontend
npm install        # 首次执行
npm run dev        # 开发模式，默认 http://localhost:5173
npm run build      # 生产构建（必须成功，dist/ 产物按路由分包）
```

启动成功标志：终端输出 `VITE vX ready` 与 `Local: http://localhost:5173/`。

## 二、代理验证

`vite.config.js` 将 `/api/user→8081`、`/api/recipe→8082`、`/api/kitchen→8083`、`/api/search→8084`、`/api/social→8085`、`/api/ai→8086`。
验证方法：浏览器开发者工具 Network 中任意接口请求应为 `http://localhost:5173/api/...` 且返回统一响应体 `{code:0,...}`。

## 三、页面功能用例

### 3.1 登录/注册（/login）
| 用例 | 操作 | 预期 |
|---|---|---|
| 注册成功 | 切到注册，填用户名/密码/昵称提交 | Toast 成功，自动登录跳首页 |
| 重复注册 | 用同一用户名再注册 | Toast 显示业务冲突提示（40900） |
| 登录失败 | 错误密码 | 错误 Toast，停留本页 |
| 演示账号 | 用户名 `demo` 登录（dev 种子） | 登录成功 |

### 3.2 首页（/）
- 推荐信息流加载菜谱卡片（封面、菜系、难度、热量、标签）；点击卡片进入详情。
- 未启动搜索服务（8084）时推荐区应有友好的空态/错误提示而非白屏。

### 3.3 搜索（/search）
- 关键词搜索（如「鸡」）返回列表；热搜 TOP 展示并可点选。
- 冰箱清理模式：按食材（逗号分隔）搜索；按最大热量筛选（如 ≤500 kcal）。
- 已登录时产生搜索历史，可清空。

### 3.4 菜谱详情（/recipe/:id）
- 顶部大图、热量环形图与三大营养素占比、适宜/慎用人群标签。
- 五步法分阶段展示（准备/洗/切/煮/装盘）+ 阶段权重进度示意。
- 「智能替换」选择场景（减脂/控糖/素食/低嘌呤）→ 显示替换方案与「减少 XX 大卡」动效。
- 「开始烹饪」：未登录跳 /login；已登录创建会话并跳厨房模式。

### 3.5 厨房模式（/cooking/:sessionId）★ 核心
| 用例 | 操作 | 预期 |
|---|---|---|
| 进度与阶段 | 进入页面 | 顶部进度条按权重 10/5/15/60/10 推进；五阶段印章指示器当前阶段朱砂高亮 |
| 大字步骤 | 查看任意步骤 | 标题特大字、详情大字，远距离可读 |
| 翻页 | 点「下一步/上一步」或左右滑动 | 步骤切换；跨阶段回退 Toast 提示「不能跨阶段回退」（40900） |
| 火候 | 进入煮（COOK）阶段 | 显示 大火🔥🔥🔥/中火🔥🔥/小火🔥 徽标 |
| 多计时器 | 在含计时步骤点「开始计时」，再开第二个 | 底部面板并行倒计时；到点震动+提示音+Toast |
| 语音指令 | 点麦克风说「下一步」 | 解析为 NEXT_STEP 并翻页；不支持语音的浏览器降级为文本输入 |
| 防误触 | 点右上角锁定 | 出现遮罩，长按 2 秒解锁 |
| 屏幕常亮 | 支持 wakeLock 的浏览器 | 页面期间屏幕不熄灭（不支持时静默） |
| 完成 | 最后一步点下一步 | 成就卡片：菜名、热量、「相当于慢跑 X 分钟」；已登录自动记录饮食；可跳打卡发帖 |
| 退出 | 点退出 | 确认后放弃会话返回 |

### 3.6 社区（/community）与发帖（/community/post）
- 作品墙按 postType 筛选，触底加载更多；点赞 toggle 数字即时变化；卡片浮层看图与评论、可发评论。
- 发帖：多图 URL 预览、关联菜谱预填（query recipeId）；提交成功后自动打卡，弹「连续打卡 X 天」与新徽章动效。

### 3.7 个人中心（/profile）与 UGC 上传（/profile/ugc）
- 健康档案查看/编辑（身高体重/过敏史/偏好/热量目标）；标签增删。
- 饮食日历：月视图每日总热量、超标标红；点某天看热量环形图与餐次明细。
- 打卡日历与徽章墙（未获得置灰）。
- UGC 上传五步向导：基本信息→食材（实时估算总热量）→五阶段步骤（五阶段各 ≥1 条才能提交，COOK 阶段可设计时与火候）→标签→预览提交，成功提示「已提交审核」。

## 四、兼容性与响应式测试

| 场景 | 方法 | 预期 |
|---|---|---|
| 移动端 H5 | DevTools 切 iPhone/Android 视口（375px） | 单列布局、底部 TabBar、触摸滑动可用 |

## 五、构建与性能验证

- `npm run build` 必须成功；产物按路由懒加载分包（9 个页面各自 chunk）。
- `npm run preview` 启动产物预览，首屏加载应只拉取首页相关 chunk。
- Lighthouse（移动端）建议 Performance ≥ 85。

## 六、常见问题

| 现象 | 原因与处理 |
|---|---|
| 接口全部报错 | 对应后端服务未启动；按 README 启动顺序启动 8081~8086 |
| 登录后刷新丢失 | localStorage 被清空；重新登录 |
| 语音按钮无反应 | 浏览器不支持 webkitSpeechRecognition，使用降级文本输入 |
| 计时器到点无震动 | 桌面浏览器无 vibrate API，属正常，以提示音与 Toast 为准 |
